# K1 學習平台 — 製作流程

**原則：**
- 每個階段必須完成並確認，先至進行下一個
- 已建立嘅嘢必須實用、嚴謹
- 家長可以全程監督

---

## 📋 製作階段總覽

```
Phase 0 ─── 確認框架 ────────────────── ✅ 完成
    │
    ▼
Phase 1 ─── 技術架構 ──────────────────→ 你確認
    │    ├── HTML/CSS/JS 結構
    │    ├── 活動引擎（點擊 + 拖拉）
    │    └── 家長資訊（i）系統
    │
    ▼
Phase 2 ─── Week 1 活動設計 ──────────→ 你確認
    │    ├── 5個 Session 活動
    │    ├── 每個活動的家長資訊
    │    └── 音頻腳本（如需要）
    │
    ▼
Phase 3 ─── Week 1 開發 ──────────────→ 你確認
    │    ├── 10個活動（代碼）
    │    ├── 家長資訊顯示
    │    └── 進度儲存
    │
    ▼
Phase 4 ─── Week 1 測試 ──────────────→ 你確認
    │    ├── 單元測試
    │    ├── 用戶測試（真實小朋友）
    │    └── 優化調整
    │
    ▼
Phase 5 ─── Week 2-4 擴展 ───────────→ 重複 Phase 2-4
    │
    ▼
Phase 6 ─── 家長Dashboard ────────────→ 你確認
```

## 🔍 每個階段品質標準

### Phase 1：技術架構

**交付物：**
- [ ] `index.html` — 主頁面框架
- [ ] `css/style.css` — 樣式系統
- [ ] `js/app.js` — 主應用邏輯
- [ ] `js/activities.js` — 活動引擎
- [ ] `js/progress.js` — 進度管理

**品質標準：**
- 所有按鈕可點擊
- 動畫流暢（60fps）
- 響應式設計（iPad + 手機）
- 無 JavaScript 錯誤

**確認問題：**
1. 主頁面嘅科目按鈕work唔work？
2. 活動引擎可以正常切換活動類型？
3. 家長資訊（i）按鈕可以正常彈出？

---

### Phase 2：Week 1 活動設計

**交付物：**
- [ ] Week 1 詳細活動設計（每個活動）
- [ ] 每個活動的家長資訊卡
- [ ] 活動的 JSON 定義
- [ ] 所需素材清單（圖片、音頻）

**品質標準：**
- 每個活動有清晰嘅學習目標
- 每個活動有發展心理學理論支持
- 每個活動有家長延伸應用建議
- 活動難度適合 K1 年齡

**確認問題：**
1. 你覺得呢個活動適合你小朋友？
2. 家長資訊嘅解釋清唔清楚？
3. 有冇遺漏某個重要嘅學習點？

### Phase 3：Week 1 開發

**交付物：**
- [ ] 10個活動代碼（可運行）
- [ ] 所有家長資訊卡
- [ ] 進度儲存功能
- [ ] 簡單的動畫效果

**品質標準：**
- 每個活動可完整運行
- 成功/失敗反饋正常
- 家長資訊可正常顯示/關閉
- LocalStorage 可正確保存進度

**確認問題：**
1. 活動係咪可以正常玩？
2. 個UI係咪適合小朋友？
3. 動畫/反饋係咪合適？

---

### Phase 4：Week 1 測試

**交付物：**
- [ ] 單元測試結果
- [ ] 用戶測試報告（真實小朋友）
- [ ] 問題清單 + 修復記錄
- [ ] 最終優化版本

**品質標準：**
- 真實小朋友可以獨立完成活動
- 成功率 > 80%
- 0 個明顯問題（crash、error）
- 家長滿意度 > 4/5

**確認問題：**
1. 小朋友係咪鍾意呢個活動？
2. 小朋友可唔可以自己操作？
3. 你作為家長，係咪滿意呢個體驗？

## 🚀 Phase 1 詳細任務

### 1.1 建立檔案結構

```
grow-up/
├── index.html           ← 主頁面
├── css/
│   └── style.css       ← 所有樣式
├── js/
│   ├── app.js          ← 主邏輯、路由
│   ├── activities.js    ← 活動引擎
│   └── progress.js     ← 進度管理
├── data/
│   └── week01.json     ← Week 1 活動定義
└── img/                ← 圖片素材
```

### 1.2 主頁面功能

- [ ] 顯示：「星期X」（Day 1-5）
- [ ] 每個 Day 有 2-4 個活動
- [ ] （i）按鈕可以睇家長資訊
- [ ] 簡單的返回按鈕

### 1.3 活動引擎

**支援活動類型：**
- [ ] Tap to Choose（點擊選擇）
- [ ] Drag & Drop（拖拉配對）

**每個活動需要：**
- [ ] 語音提示
- [ ] 選項顯示（2-4個）
- [ ] 成功反饋動畫
- [ ] 失敗反饋（溫和）
- [ ] 自動下一題（成功後）

### 1.4 家長資訊系統

- [ ] （i）按鈕在每個活動旁邊
- [ ] 點擊後彈出資訊卡
- [ ] 包含：學習目標、設計理念、心理學、延伸建議
- [ ] 可以關閉

### 1.5 進度管理

- [ ] LocalStorage 保存進度
- [ ] 標記已完成嘅活動
- [ ] 顯示完成狀態
