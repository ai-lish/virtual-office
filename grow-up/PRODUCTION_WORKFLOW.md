# K1 學習平台 — 製作流程

**原則：**
- 每個階段必須完成並確認，先至進行下一個
- 已建立嘅嘢必須實用、嚴謹
- 家長可以全程監督
- **Audio-first**：K1 小朋友唔識字，所有指示必須有語音
- **大按鈕、大目標**：觸控目標最少 80×80px，建議 100×100px（配合 K1 幼兒需要）
- **溫和反饋**：永遠唔會話「錯」，只會鼓勵再試

## 🔍 每個階段品質標準

### Phase 1：技術架構

**交付物：**
- [ ] `index.html` — 主頁面框架
- [ ] `css/style.css` — 樣式系統
- [ ] `css/animations.css` — 動畫效果（獎勵、過場）
- [ ] `js/app.js` — 主應用邏輯
- [ ] `js/activities.js` — 活動引擎
- [ ] `js/progress.js` — 進度管理

**品質標準：**
- 所有按鈕可點擊（touch + click 雙重支援）
- 動畫流暢（60fps）
- 響應式設計（iPad + 手機）
- 無 JavaScript 錯誤
- 觸控目標最少 80×80px，建議 100×100px

**確認問題：**
1. 主頁面嘅科目按鈕work唔work？
2. 活動引擎可以正常切換活動類型？
3. 家長資訊（i）按鈕可以正常彈出？

### Phase 1.5：素材準備計劃

**交付物：**
- [ ] 圖片素材來源方案（手繪 / AI 生成 / 免費素材庫）
- [ ] 音頻錄製方案（TTS / 真人錄音）
- [ ] Week 1 所需素材完整清單
- [ ] 素材命名規範

**品質標準：**
- 圖片風格統一、色彩鮮明
- 音頻語速適合 3-4 歲（慢、清晰、廣東話）
- 所有素材有清晰嘅版權狀態

> ⚠️ 素材製作往往係最大瓶頸，提早規劃可以避免 Phase 3 卡住

### Phase 2：Week 1 活動設計

**交付物：**
- [ ] Week 1 詳細活動設計（每個活動）
- [ ] 每個活動的家長資訊卡
- [ ] 活動的 JSON 定義
- [ ] 所需素材清單（圖片、音頻）
- [ ] 音頻腳本（每句對白）

**品質標準：**
- 每個活動有清晰嘅學習目標
- 每個活動有發展心理學理論支持
- 每個活動有家長延伸應用建議
- 活動難度適合 K1 年齡
- 每個 Session 唔超過 10-15 分鐘（K1 注意力上限）

**確認問題：**

2. 家長資訊嘅解釋清唔清楚？
3. 有冇遺漏某個重要嘅學習點？

**交付物：**
- [ ] 10個活動代碼（可運行）

- [ ] 進度儲存功能
- [ ] 獎勵動畫效果（星星、貼紙）

**品質標準：**
- 每個活動可完整運行
- 成功/失敗反饋正常（失敗 = 鼓勵，唔係懲罰）
- 家長資訊可正常顯示/關閉

- Touch events + click events 都 work

**確認問題：**
1. 活動係咪可以正常玩？
2. 個UI係咪適合小朋友？
3. 動畫/反饋係咪合適？

---

### Phase 4：Week 1 測試

**交付物：**
- [ ] 設備測試結果（iPad Safari、Android Chrome、手機）
- [ ] 用戶測試報告（至少 2-3 個真實小朋友）
- [ ] 使用時間記錄（注意力數據）
- [ ] 問題清單 + 修復記錄
- [ ] 最終優化版本

**品質標準：**
- 真實小朋友可以獨立完成活動（至少測試 2-3 個小朋友）
- 成功率 > 80%（定義：唔需要家長幫手就完成嘅活動比例）
- 平均使用時間記錄（超過 15 分鐘要加休息提示）
- 0 個明顯問題（crash、error）
- 多設備測試通過（iPad Safari、Android Chrome、手機）
- 家長滿意度 > 4/5

**確認問題：**
1. 小朋友係咪鍾意呢個活動？
2. 小朋友可唔可以自己操作？
3. 你作為家長，係咪滿意呢個體驗？

**回退標準：**
> 如果測試揭示根本性 UX 問題（例如 Drag & Drop 太難、選項太多），回退至 Phase 2 重新設計，唔好強行修補

## 🚀 Phase 1 詳細任務

### 1.1 建立檔案結構

```
grow-up/
├── index.html           ← 主頁面
├── css/
│   ├── style.css       ← 所有樣式
│   └── animations.css  ← 動畫效果（獎勵、過場）
├── js/
│   ├── app.js          ← 主邏輯、路由
│   ├── activities.js    ← 活動引擎
│   └── progress.js     ← 進度管理
├── data/
│   ├── schema.json     ← 活動 JSON schema 定義
│   └── week01.json     ← Week 1 活動定義
├── audio/              ← 語音指示、音效
└── img/                ← 圖片素材
```

### 1.2 主頁面功能

- [ ] 顯示：「星期X」（Day 1-5）
- [ ] 每個 Day 有 2-4 個活動
- [ ] （i）按鈕可以睇家長資訊
- [ ] 簡單的返回按鈕

### 1.3 活動引擎

**支援活動類型：**
- [ ] Tap to Choose（點擊選擇）— 主要互動方式
- [ ] Drag & Drop（拖拉配對）— 需要超大放置區域 + 磁吸效果（K1 精細動作仲發展緊）

**每個活動需要：**
- [ ] 語音提示（必須，K1 唔識字）
- [ ] 選項顯示（2-3個，唔好太多）
- [ ] 成功反饋動畫（星星、拍手、彩色效果）
- [ ] 失敗反饋（溫和 —「再試吓！」配鼓勵音效，絕不用 buzzer）
- [ ] 自動下一題（成功後，停 1.5-2 秒俾小朋友睇到成功）
- [ ] 觸控 + 點擊雙重支援（touch events + click events）

**注意力管理：**
- [ ] 每個 Session 唔超過 10-15 分鐘（K1 注意力上限）
- [ ] 中間自動建議休息（「做得好！休息一陣？」）

### 1.4 家長資訊系統

- [ ] （i）按鈕在每個活動旁邊
- [ ] 點擊後彈出資訊卡
- [ ] 包含：學習目標、設計理念、心理學、延伸建議
- [ ] 可以關閉

### 1.5 進度管理

- [ ] LocalStorage 保存進度
- [ ] 標記已完成嘅活動
- [ ] 顯示完成狀態（星星/貼紙獎勵系統）
- [ ] 進度匯出功能（JSON download，防止清 browser data 失去進度）

> ⚠️ 已知限制：LocalStorage 會因清除瀏覽器資料而失去進度，家長需知悉
