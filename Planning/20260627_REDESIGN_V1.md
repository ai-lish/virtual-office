## 6. Risks & open questions

### 6.1 Risks

1. **`fetch-usage.py` 改動風險**（Phase 3）
   - Risk: 改 Python script 可能 break cron 流程
   - Mitigation: 先測試本地 run `fetch-usage.py`，確認生成 JSON 正確，先至 commit
   - Severity: Medium
   - Impact: 如果 break 咗，cron 流程會停止生成 `usage-quota.json`，但舊 JSON 仍然可讀
   - Rollback plan: revert `fetch-usage.py` 改動，前端改用 workaround（自己計 `used_percent`）

2. **Dashboard `tab-history` 太長**（Phase 2）
   - Risk: 將 AI Coding + MiniMax v2 + v1 擺晒落一個 tab，可能太長
   - Mitigation: 用 collapsible section 或 pagination
   - Severity: Low
   - Impact: UX 稍差，但唔影響功能
   - Rollback plan: 將 AI Coding 歷史（`usage-history.json` 15 個 snapshot）延後到日後 planning

3. **`usage-quota.json` top-level `used_percent` field backward compatible**
   - Risk: Dashboard 現時 `aitoolsRenderProvider` 依賴 `data.used_percent` top-level field，但舊 JSON 冇呢個 field
   - Mitigation: 前端改用 fallback logic：`data.used_percent || max(data.primary_5h.used_percent, data.secondary_7d.used_percent)`
   - Severity: Low
   - Impact: 舊 JSON 仍然可讀
   - Rollback plan: 唔使 rollback，fallback logic 已經處理

4. **Responsive design mobile 堆疊**
   - Risk: 手機版 Codex + Claude card 堆疊後，history tab 太長
   - Mitigation: 用 collapsible section 或 default collapse Codex + Claude card
   - Severity: Low
   - Impact: UX 稍差，但唔影響功能
   - Rollback plan: 將 AI Coding 移返去獨立 tab（revert Phase 2）
